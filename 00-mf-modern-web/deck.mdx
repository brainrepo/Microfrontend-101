# Single-Page Application


Una single page application è un'app che funziona all'interno di un browser 
e che non richiede il ricaricamento della pagina
durante l'uso.

---

<img src="https://echoinnovateit.com/wp-content/uploads/2020/02/SPAvsMPA.png" style={{"width": "40vw"}}/>

---

<Split>

- nessun reload della pagina
- nessun tempo di attesa extra (js css html)

</Split>

<Split>

<img src="https://media.giphy.com/media/3otPoNMCHgEfLaO1NK/giphy.gif" style={{"width": "40vw"}} />

SPA richiede il markup e i dati in modo indipendente e rende le pagine direttamente nel browser.

</Split>


---

# Vantaggi della SPA:

- Veloce
  - HTML + CSS + script vengono caricati solo una volta
  - Solo i dati vengono trasmessi avanti e indietro.

---

# Vantaggi della SPA:

### Lo sviluppo è semplificato <br />  e ottimizzato. 

- Non è necessario scrivere codice per eseguire il rendering delle pagine sul server.
- È molto più facile iniziare perché di solito è possibile avviare lo sviluppo da un file senza utilizzare alcun server.

---

# Vantaggi della SPA

### Le SPA sono facili <br /> da eseguire il debugger di Chrome

- monitorare le operazioni di rete
- esaminare elementi di pagina e dati ad essa associati.

---

# Vantaggi della SPA

### È più facile creare <br /> un'applicazione mobile 

lo sviluppatore può riutilizzare lo stesso codice di backend per l'applicazione web e l'applicazione mobile nativa.

---

# Vantaggi della SPA

### SPA può memorizzare in modo <br /> efficace qualsiasi memoria locale. 

Un'applicazione invia una sola richiesta, memorizza tutti i dati, <br/>
quindi può utilizzare questi dati e funziona anche offline

---

# Contro delle Single-Page Application:


È molto complicato fare l'ottimizzazione SEO di un'applicazione a pagina singola.

Il suo contenuto viene caricato da AJAX 

(anche se gli algoritmi di google indicizzano i contenuti <br/>
ajax caricati nei primi secondi di loading della pagina) 

---

# Contro delle Single-Page Application:

È lenta da scaricare perché è necessario scaricare **client framework pesanti**.

Richiede che JavaScript sia presente e abilitato. 
(rendering isomorfo / lato server è già possibile eseguire il rendering della pagina sul server)

---

# Contro delle Single-Page Application:

Rispetto all'applicazione “tradizionale”, SPA è meno sicura a casa della possibilità di verificarsi del Cross-Site Scripting (XSS)
consente agli aggressori di iniettare script lato client nell'applicazione Web da parte di altri utenti.

---

# Contro delle Single-Page Application:

Il memory leak in JavaScript può persino rallentare il sistema più potente (guardare la memoria della webapp di facebook)

# Multi-Page Application

Le applicazioni a più pagine funzionano in modo "tradizionale". 

Ogni cambiamento rendendo una nuova pagina dal server nel browser. 

</Split>


---

# Pro delle Multiple-Page Application

- Struttura solida ogni pagina è un nodo della Struttura
- Molto buona e facile per una corretta gestione SEO. 

---

# Contro dell'applicazione multipagina:

Lo sviluppo del frontend e del backend sono strettamente collegati.

# Installabile

- Girano in una finestra autonoma anziché in una scheda del browser. 
- Possono essere avviati dalla schermata iniziale dell'utente, dal dock, dalla barra delle applicazioni. 
- Le scorciatoie da tastiera solitamente riservate durante l'esecuzione nel browser diventano disponibili. 

--- 

Le Progressive Web Apps possono registrarsi per accettare il contenuto da altre applicazioni o per essere 
l'applicazione predefinita per gestire diversi tipi di file.

--- 

# Caratteristiche

---

#Reattive: 

per adattarsi a qualsiasi fattore di forma

---

#Indipendenti dalla connettività: 

progressive enhanced con i Service Worker per consentire loro di lavorare offline

---

# Interazioni simili ad app: 

adotta un modello di applicazione Shell + Content per creare navigazioni e interazioni appy

---

# Fresco: 

Trasparentemente sempre aggiornato grazie al processo di aggiornamento del Service Worker

---

# Sicuro: 

servito tramite TLS (un requisito del Service Worker) per prevenire lo snooping (metodo di intercettazione del traffico multicast. )

---

# Rilevabili: 

sono identificabili come "applicazioni" grazie ai manifesti del W3C 

---

# Ri-ingaggianti: 

Notifiche push

---

# Installabile: 

sulla schermata iniziale tramite prompt forniti dal browser, 
consentendo agli utenti di "conservare" le app che trovano più utili senza il fastidio di un app store

---

# Collegabile: 

significa che sono zero attriti, zero installazione e facili da condividere. 
Il potere sociale degli URL è importante.
